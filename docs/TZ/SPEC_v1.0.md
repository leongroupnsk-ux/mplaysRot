# Техническое задание: Интеграция рекламных кабинетов и AI/BI-систем в платформу Attribly

**Версия 1.0**  
**Дата:** 6 мая 2026 г.  
**Статус:** Активное развитие (Phase 0 – Planning)

---

## 1. Общие сведения

**Наименование:** Модуль интеграции внешних рекламных кабинетов и AI/BI-систем.  
**Кодовое имя:** `attribly-ads-ai`.  
**Назначение:** Обеспечить в рамках платформы Attribly полный цикл управления внешним трафиком — от сбора статистики рекламных кабинетов до интеллектуальной атрибуции и прогнозирования.  

**Границы:** Модуль включает интеграционные адаптеры к:
- Яндекс.Директ
- VK Ads  
- Telegram Ads
- VK Блогер
- OpenAI-совместимый API
- Собственные ML-модели атрибуции
- Экспорт данных в BI-системы

---

## 2. Цели и задачи

1. **Централизация управления** — рекламными кампаниями селлеров из единого интерфейса Attribly.
2. **Автоматический сбор статистики** — расходы, показы, клики, конверсии по каждому подключённому кабинету.
3. **Механизм обогащения данных** — сквозная атрибуция для точного расчёта ROMI.
4. **ИИ-ассистент** — рекомендации по логистике и маркетингу на базе OpenAI API.
5. **ML-модель атрибуции** — вероятностная модель на CatBoost для Wildberries (и Amazon).
6. **Выгрузка данных для BI** — сырые данные для самостоятельного анализа.

## 3. Функциональные требования

### 3.1 Интеграция с Яндекс.Директ

**Подключение**
- OAuth 2.0 авторизация (scope: `direct_api`, `audience`)
- Шифрование токенов AES-256-GCM
- Автоматическое обновление refresh-токенов

**Сбор статистики**
- Ежедневная синхронизация по крону
- API v5: `reports` — статистика по кампаниям, группам, объявлениям, ключевым словам
- Поля: `Impressions`, `Clicks`, `Cost`, `Conversions`, `Revenue`
- Таблица: `ad_performance` (campaign_id, user_id, date)

**Управление аудиториями**
- Чтение сегментов через `audience` API
- Создание look-alike сегментов на основе Attribly seed
- Загрузка обезличенных идентификаторов (хеш email/телефон, Yandex ID)

### 3.2 Интеграция с VK Ads

**Подключение**
- OAuth 2.0 (scope: `ads_api`, `retargeting`)
- Аналогичное хранение токенов

**Сбор статистики**
- Метод `statistics`: spent, impressions, clicks, ctr, conversions
- Демографические разрезы (возраст, гео, устройства)

**Управление аудиториями**
- Upload seed-сегментов (VK ID, IMEI/GID хешированные)
- Активация look-alike с отслеживанием статуса

**Товарные фиды**
- Синхронизация каталога в CSV/XML через API фидов

### 3.3 Интеграция с Telegram Ads

**Подключение**
- Official Telegram Ad Platform API
- Генерация Pixel Tag

**Сбор статистики**
- spent, impressions, clicks
- Конверсии через Attribly Pixel (связка с `trax_id`)

**Отслеживание событий**
- Пиксель на прокси-лендингах
- События: click, add_to_cart, purchase
- Атрибуция через utm_source и pixel_id

### 3.4 Интеграция с VK Блогер

**Подключение**
- Official VK Блогер API (token-based)

**Сбор статистики**
- Охваты, вовлечённость, переходы по уникальной ссылке
- Сопоставление с `trax_id` и UTM-метками

**Автоматизация**
- Создание заказов на размещение
- Оценка эффективности блогера по доходу

### 3.5 Интеграция AI (OpenAI-совместимый API)

**Подключение**
- API-ключ, endpoint, модель в разделе «Интеграции»
- Шифрование ключей
- Опциональный прокси-эндпоинт Attribly для контроля

**ИИ-ассистент**
- Контекстная подсказка с актуальными данными (остатки, продажи, статистика)
- Свободный текст вопроса
- Markdown-ответ в интерфейсе
- Ограничения по тарифам: 50/мес (Business), безлимит (Enterprise)
- Логирование запросов/ответов

**Технические детали**
- Библиотека `openai`
- Модель: gpt-4o (настраиваемая)
- Температура: 0.3
- Таймаут: 15 секунд
- Кеширование в Redis (идентичные контексты)

### 3.6 ML-модель атрибуции Wildberries

**Алгоритм**
- CatBoostClassifier: iterations=500, learning_rate=0.03, depth=6, eval_metric=AUC
- Переобучение: один раз в сутки на подтверждённых данных

**Признаки (фичи)**
- `time_diff` — разница в минутах между кликом и заказом
- `geo_match` — совпадение региона (бинарный)
- `device_match` — совпадение устройства (бинарный)
- `source_historical_conv_rate` — средняя конверсия за 30 дней
- `clicks_from_same_ip` — количество кликов с этого IP за час
- `order_amount` — сумма заказа
- `day_of_week`, `hour_of_day` — временные признаки

**Выход модели**
- Вероятность `confidence` [0, 1]
- Порог автоматической атрибуции: 0.7
- <0.7 → "требующий ручной верификации"

**Интеграция**
- Формат: ONNX
- При поступлении заказа WB: поиск подходящих кликов за 24 часа
- Наивысшая вероятность — атрибуция

**Дообучение**
- Пользователи подтверждают/отклоняют атрибуцию
- Подтверждённые связки → обучающий набор
- Ежедневное переобучение автоматически + ручной запуск из админки

### 3.7 Интеграция с BI-системами

**Выгрузка сырых данных**
- Экспорт CSV из раздела «Атрибуция» и «Логистика»
- Поля атрибуции: trax_id, timestamp_click, timestamp_order, order_id, sku, revenue, source, campaign, confidence, verified
- Поля логистики: warehouse_id, nm_id, chrt_id, size, stock_qty, sales_7d, returns_7d, return_rate, days_of_supply

**Реализация**
- Кнопка «Экспорт CSV» в интерфейсе
- Запрос: `POST /analytics/export`
- Для >10K строк — фоновая задача Celery с email-уведомлением

**Roadmap (прямые коннекторы)**
- Power BI, Tableau коннекторы
- ODBC/JDBC-доступ к ClickHouse (Enterprise)

## 4. Техническая архитектура

### 4.1 Микросервисная структура

- **Integration Adapters** — Яндекс.Директ, VK Ads, Telegram Ads, VK Блогер (OAuth, статистика, аудитории)
- **ML Service** — инференс и обучение CatBoost моделей
- **AI Assistant Service** — прокси OpenAI с контролем лимитов
- **Export Service** — CSV и BI-интеграции

### 4.2 Хранение данных

**PostgreSQL**
- integrations
- ad_campaigns
- ad_performance
- ml_models
- ai_queries
- attributed_orders

**ClickHouse**
- tracking_events (быстрая аналитика)
- attributed_orders

**Redis**
- OAuth-токены (кеширование)
- Результаты look-alike
- AI-ответы (кеш)

### 4.3 Безопасность

- Все секреты шифруются
- Логирование административных действий
- Rate limiting на AI-запросы
- JWT-аутентификация для всех эндпоинтов

## 6. API-контракты (основные эндпоинты)

| Метод | Путь | Описание | Auth |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/integrations/{provider}/auth` | Запуск OAuth flow | JWT |
| `GET` | `/api/v1/integrations/{provider}/status` | Проверка статуса подключения | JWT |
| `GET` | `/api/v1/analytics/performance` | Сводная статистика по всем кабинетам | JWT |
| `POST`| `/api/v1/audiences/lookalike` | Создание и активация look-alike сегмента | JWT |
| `POST`| `/api/v1/ai/ask` | Запрос к ИИ-ассистенту | JWT |
| `POST`| `/api/v1/export/attribution` | Экспорт данных атрибуции | JWT |
| `POST`| `/api/v1/export/logistics` | Экспорт логистических данных | JWT |

All endpoints require JWT auth. Admin endpoints require `admin` role.
